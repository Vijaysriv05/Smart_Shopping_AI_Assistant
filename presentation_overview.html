Project Presentation Guide: Goval AI - Smart Autonomous Shopping AI Agent

This guide outlines the features, technology stack, AI details, prompt engineering techniques, and key unique selling points (USPs) of Goval AI to help you deliver an outstanding project presentation.

1. Core Technology Stack

Your project is built using a modern, scalable, decoupled full-stack architecture:
  • Frontend:
  • React (Vite + TypeScript): Provides a fast, type-safe, single-page application framework.
  • Tailwind CSS: A utility-first CSS framework for custom responsive styling.
  • Framer Motion & Lucide React: Delivers smooth, premium micro-animations (e.g., page transitions, interactive hover effects) and modern iconography.
  • React Query (TanStack): Manages state caching, backend synchronization, and query loading states.
  • Backend:
  • Java Spring Boot: A production-ready REST API gateway built using Spring Web, Spring Security, and Maven dependency management.
  • Spring Data JPA (Hibernate): Simplifies database interactions, query generation, and object-relational mapping (ORM).
  • Database:
  • MySQL: A robust relational database storing product catalogs, user session tracking, shopping carts, order histories, wishlists, and spending analytics records.

2. Artificial Intelligence & LLM Details

  • LLM Model Used:
  • Google Gemini (gemini-2.0-flash and gemini-1.5-flash): Accessed via the Google AI Studio developer endpoint. The backend implements a multi-model failover mechanism: if gemini-2.0-flash fails or hits rate limits, it automatically falls back to gemini-1.5-flash.
  • Type of AI Implemented:
  • Orchestrated Agentic AI: The system functions as an autonomous shopping helper. It doesn't just chat—it tracks session context, constructs personalized profiles, calculates budget splits, compares catalog item specs, and computes potential savings.
  • Deterministic Hybrid Heuristics: To ensure 100% service availability, the backend is built with a dual-layer AI architecture. If Gemini is rate-limited (HTTP 429), a local keyword-matching algorithm dynamically executes (tokenizing the goal title, mapping categories, sorting by rating, and generating cheaper alternatives) so the user never sees a broken interface.

3. Prompt Engineering Techniques Used

The project utilizes advanced prompt engineering to achieve reliable backend integration:

1. System Persona Prompting:

  • Instructs Gemini to assume specific roles (e.g., *"You are an expert AI Shopping Goal Planner"*, *"You are an expert product analyst"*).
2. Structured JSON Mode:
  • Prompts explicitly specify a strict JSON schema and demand a raw JSON string output (using responseMimeType: "application/json"). This eliminates natural language filler text, allowing the backend to deserialize the model's response directly into Java objects using Jackson ObjectMapper.
3. In-Context Learning (RAG-style Catalog Injection):
  • Instead of letting Gemini recommend non-existent items, the backend dynamically queries the MySQL database and injects the live product catalog (IDs, names, brands, categories, prices, ratings) directly into the prompt context. This forces Gemini to align its recommendations with real catalog items.

4. Key Project Features

  • Conversational Shopping Assistant: Users can ask natural language queries to receive product suggestions, budget evaluations, and reasoning explanations.
  • Smart Shopping Goal Planner: Automatically takes a user's goal (e.g., "Joining College") and budget, then constructs a categorized budget allocation plan (Must Have, Highly Recommended, Nice to Have).
  • Side-by-Side Comparison Engine: Compares multiple items detailing specs, ratings, and displays automated Gemini Choice Awards (e.g., Value Pick, Premium Pick, Top Spec).
  • Interactive Personal Dashboard: Displays monthly, weekly, and yearly AI Spending Reports, budget utilization trackers, and coupon savings history.
  • Real-time Cart & Order Checkout: Saves and confirms orders, which dynamically update the dashboard statistics.

5. Project Uniqueness & Competitive Advantage

  • Dual-Layer Fault Tolerant Architecture: Unlike other AI apps that crash when API keys hit free-tier rate limits, Goval AI's backend detects the failure and dynamically generates fallback category allocations and cheaper recommendations using local database heuristics.
  • Value-Focused Alternative Suggestion: The planner actively analyzes items and suggests cheaper database alternatives in the same category, showing the exact savings (e.g., *“Replace Product A with Product B to save $15.00”*).
  • Granular Budget Blueprints: Rather than just displaying list recommendations, it performs actual financial blueprinting, splitting the budget into percentages mapped to priorities.
  • Zero-Login Personalization: Utilizes lightweight, dynamic session-based tracking (x-session-id) so users receive a tailored, high-fidelity experience without onboarding friction.

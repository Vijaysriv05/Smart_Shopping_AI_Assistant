# Smart Autonomous Shopping Assistant using Agentic AI

This project is a complete, production-ready **Agentic AI Shopping Assistant** built with a React + TypeScript frontend, a Node.js Express backend using Drizzle ORM, and a MySQL database. It utilizes the **Google Gemini API** to autonomously reason, compare products, plan budgets, analyze user sentiment, detect fake/suspicious reviews, predict pricing trends, and maintain long-term memory of user conversations.

---

## ─── Project Objective ──────────────────────────────────────────

Unlike traditional e-commerce platforms that place the burden of searching, filtering, and decision-making on the user, this platform acts as an **autonomous shopping companion**. 

The AI agent:
1. **Extracts Requirements**: Learns user preferences, preferred brands, category interests, and budget limits.
2. **Reasons & Explains**: Explains *why* a product is recommended rather than showing generic rankings.
3. **Monitors Catalog**: Detects overpriced items, recommends bundles, identifies similar duplicates, alerts for stock shortages, and tracks price histories.
4. **Maintains Memory**: Recalls previous chats and preferences across sessions.

## ─── Database Schema ─────────────────────────────────────────

The relational MySQL database schema is managed via **Drizzle ORM**. Below is the table structure:

### 1. `categories` Table
* `id` (int, PK, autoincrement)
* `name` (text, notNull) - Category name (e.g. Laptops, Smartphones)
* `slug` (text, notNull) - URL-friendly category slug
* `icon` (text) - Lucide icon reference string

### 2. `products` Table
* `id` (int, PK, autoincrement)
* `name` (text, notNull) - Product name
* `brand` (text, notNull) - Manufacturer brand
* `category` (text, notNull) - Associated category name
* `price` (decimal, notNull) - Product current price in cents
* `originalPrice` (decimal) - Product catalog original price (pre-discount) in cents
* `imageUrl` (text) - Unique Unsplash image URL
* `description` (text, notNull) - Detail description of product
* `specs` (json) - Key-value pair specification details
* `rating` (decimal, default 0.0) - High-resolution average rating (e.g. 4.8)
* `reviewCount` (int, default 0) - Verified user review count
* `inStock` (boolean, default true) - Stock availability flag
* `isFeatured` (boolean, default false) - Featured product flag
* `tags` (json) - Custom tags array
* `aiScore` (decimal) - Autonomous score generated by Gemini (0-100)
* `createdAt` (timestamp, default now)

### 3. `users` Table
* `id` (int, PK, autoincrement)
* `email` (varchar, Unique, notNull) - Registration email
* `passwordHash` (text, notNull) - BCrypt encrypted password
* `name` (text, notNull) - User display name
* `role` (varchar, default "user") - Role based permissions ("user" | "admin")
* `preferences` (json) - Saved preferences dictionary
* `createdAt` (timestamp, default now)

### 4. `cart` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull) - Session-tied identifier
* `productId` (int, notNull) - Foreign reference to `products`
* `quantity` (int, notNull, default 1)
* `addedAt` (timestamp, default now)

### 5. `wishlist` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull) - Session identifier
* `productId` (int, notNull) - Foreign reference to `products`
* `createdAt` (timestamp, default now)

### 6. `orders` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull)
* `items` (json, notNull) - Ordered items snapshot array
* `totalAmount` (decimal, notNull)
* `status` (varchar, default "pending")
* `createdAt` (timestamp, default now)

### 7. `chats` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull)
* `role` (varchar, notNull) - "user" or "model"
* `content` (text, notNull) - Message string
* `products` (json) - Snapshot array of referenced products
* `timestamp` (timestamp, default now)

### 8. `price_history` Table
* `id` (int, PK, autoincrement)
* `productId` (int, notNull)
* `price` (decimal, notNull)
* `recordedAt` (timestamp, default now)

### 9. `recommendation_history` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull)
* `query` (text, notNull) - User search query
* `reasoning` (text, notNull) - AI Agent reasoning output
* `productIds` (json, notNull) - Recommended products IDs array
* `createdAt` (timestamp, default now)

### 10. `behaviour` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull)
* `actionType` (varchar, notNull) - Tracking action (e.g. view, add_to_cart, search)
* `details` (json)
* `timestamp` (timestamp, default now)

### 11. `analytics` Table
* `id` (int, PK, autoincrement)
* `metricName` (varchar, notNull) - Metric tag name
* `metricValue` (double, notNull)
* `dimensions` (json)
* `timestamp` (timestamp, default now)

### 12. `ai_logs` Table
* `id` (int, PK, autoincrement)
* `sessionId` (text, notNull)
* `agentName` (varchar, notNull)
* `prompt` (text, notNull) - Full constructed prompt
* `response` (text, notNull) - Raw model response text
* `steps` (json) - Thought path steps
* `latencyMs` (int) - Request execution duration in milliseconds
* `timestamp` (timestamp, default now)

## ─── Prompt Engineering Documentation ─────────────────────────

The API server encapsulates raw user prompts using structured system instruction layouts.

### 1. Reusable Prompt Templates (Backend Implementation)

* **Shopping Recommendations (`/api/ai/recommend`)**:
  Combines user query, category interest, active budget parameters, chat logs history, and user favorite categories/brands. Requests a structural JSON response declaring reasoning, matching products, pros, cons, target use-cases, and follow-up prompts.
* **Product Comparison (`/api/ai/compare`)**:
  Combines specs, rating, price, and descriptions of multiple products. Prompts Gemini to score aspects (Price, Performance, Build, Features, Reliability, Future-Proof) and declare a clear product winner with side-by-side analysis.
* **Price Trend Analysis (`/api/ai/price-prediction/:productId`)**:
  Feeds Gemini the 6-month historical price log of the item. Asks the model to predict trend directions ("increasing", "decreasing", "stable"), state a confidence level, and recommend the best purchase window ("now", "wait", "buy immediately").
* **Product Scoring (`/api/ai/score/:productId`)**:
  Scores the product out of 100 on multiple technical categories (overall, performance, price, reliability, battery, popularity, futureProof) and summarizes user sentiment.

## ─── API Documentation ────────────────────────────────────────

### Authentication
* `POST /api/auth/register` - Create user account (JWT signed payload return)
* `POST /api/auth/login` - Authenticate credentials and return token
* `GET /api/auth/profile` - Get logged-in user profile details (protected)

### Catalog & Categories
* `GET /api/products` - List products with category, brand, search filters, and pricing caps
* `GET /api/products/:id` - Fetch single product specs
* `GET /api/categories` - Fetch all categories with count badges

### Cart & Wishlist
* `GET /api/cart` - Retrieve user shopping cart
* `POST /api/cart` - Add item to cart
* `PUT /api/cart` - Edit item quantity in cart
* `DELETE /api/cart` - Delete item or clear cart
* `GET /api/wishlist` - Fetch user wishlist items
* `POST /api/wishlist` - Toggle item in wishlist

### AI & Autonomous Endpoints
* `POST /api/ai/recommend` - Fetch AI recommendation matching query + user memory
* `POST /api/ai/compare` - Compare multiple products side-by-side
* `GET /api/ai/score/:productId` - Retrieve Gemini overall scoring parameters
* `GET /api/ai/price-prediction/:productId` - Get price change forecasts
* `GET /api/ai/memory` - Fetch AI personal profile preferences
* `PATCH /api/ai/memory` - Modify AI user memory goals/budgets
* `GET /api/ai/chat` - Fetch chat conversation history
* `GET /api/ai/autonomous-insights` - Fetch master analytics (Alerts, pricing audits, duplicate detections, review fake-pattern checks, bundles)

## ─── Installation Guide ──────────────────────────────────────

### Prerequisites
* **Node.js**: v20 or v22
* **pnpm**: v9 or v10
* **MySQL**: local running instance (MySQL 8 recommended)

### Setup Steps
1. **Clone the Repository**
2. **Install Dependencies**:
   ```bash
   pnpm install
   ```
3. **Configure Environment Variables**:
   Create a `.env` file or export variables:
   ```bash
   # Database url format (MySQL)
   DATABASE_URL="mysql://root:VaSo%40497%24@localhost:3306/ai_shopping_assistant"
   # Google Gemini API Key
   GEMINI_API_KEY="your-gemini-api-key"
   ```
4. **Push Schemas to MySQL**:
   ```bash
   pnpm --filter @workspace/db run push
   ```
5. **Seed the database (250 products, 25 categories)**:
   ```bash
   npx tsx scripts/src/seed-more-products.ts
   ```
6. **Start Frontend & Backend Development Servers**:
   ```bash
   # Run both in parallel
   pnpm run dev
   ```

## ─── Deployment Guide (Docker) ───────────────────────────────

This project includes a multi-stage `Dockerfile` and a `docker-compose.yml` to orchestrate production deployment easily.

### Step 1: Provide Environment Settings
Make sure your Gemini API key is exported or populated in your environment:
```bash
export GEMINI_API_KEY="YOUR_GEMINI_API_KEY"
```

### Step 2: Build & Start Containers
Run Docker Compose in the root directory:
```bash
docker-compose up --build -d
```

Docker Compose will:
1. Spin up a MySQL 8 database container (`shopping_db`).
2. Run a healthcheck on MySQL.
3. Build the shopping app container (`shopping_app`), compiling React assets and starting the Express server.
4. The application will be serving at **http://localhost:5000**.
